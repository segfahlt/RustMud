## Session 1 (prior context — reconstructed from history)

### World Model
**Category:** Architecture, Code
**Files:** `src/world/`, `src/lib.rs`, `Cargo.toml`
**Description:** Initial world model implemented. `Zone`, `Room`, `RoomRef`, `Direction` structs. Rooms have exits as `HashMap<Direction, RoomRef>`. Zones loaded from `data/zones/*.json`. `schemars` added for JSON Schema generation. `src/bin/schema.rs` binary prints the zone schema.

### Command Registry
**Category:** Architecture, Code
**Files:** `src/commands/mod.rs`, `src/commands/registry.rs`
**Description:** Command registry with `CommandDef` structs. Priority-based prefix matching: lower number = higher priority. Exact name → exact alias → prefix match with priority sort. `AmbiguousCommand` error if two same-priority commands share a prefix. Direction commands at priority 5, regular at 10, admin at 20. This resolved `s` matching both `south` and `shutdown`.

### Gateway / Game Split Networking
**Category:** Architecture, Code
**Files:** `src/bin/gateway.rs`, `src/proto.rs`, `src/main.rs`
**Description:** Two-binary architecture. Gateway holds all TCP connections on port 4000, communicates with game over Unix socket `/tmp/rustmud.sock` using newline-delimited JSON. Game loop is freely restartable — gateway re-announces clients on reconnect. `ClientInfo` stores `character_id` per socket for seamless reboot restore.

### State Save / Restore
**Category:** Code
**Files:** `src/persist.rs`, `src/main.rs`
**Description:** `WorldSave` persisted to `data/save/state.json` on shutdown/reboot. Stores `CharacterSave` (zone_id, room_id, health, max_health) keyed by character_id. Loaded on startup, used to restore player positions without re-prompting.

### Account System
**Category:** Code
**Files:** `src/persist.rs`, `src/main.rs`
**Description:** `AccountFile` with argon2id password hashing. One account, multiple characters. `CharacterFile` with stable id = `name.to_lowercase()`. Character select screen on login. Session state machine: NeedUsername → NeedPassword / NeedNewPassword → CharacterSelect → NeedCharName → Playing. Seamless reboot reconnect: gateway stores `character_id`, game restores without prompting.

### Permissions + Admin Commands
**Category:** Architecture, Code
**Files:** `src/persist.rs`, `src/commands/mod.rs`, `src/commands/registry.rs`, `src/main.rs`, `src/bin/gateway.rs`
**Description:** Non-exclusive `Permission` enum on `CharacterFile`: player, remort, builder, monitor, dev, admin. `has_perm()` — Admin satisfies any check. `SessionState::Playing` caches permissions loaded at login. Admin commands: `shutdown` (kills gateway), `reboot` (restarts game), `reboot refresh` (saves at home room, disconnects all, restarts game). `Signal` enum + `mpsc::channel` for admin commands to communicate with main loop. `GatewayMsg::DisconnectAll` and `GatewayMsg::Shutdown` added to IPC protocol.

## Session 2 — 2026-06-12

### Permissions Moved to CharacterFile
**Category:** Architecture, Code
**Files:** `src/persist.rs`, `src/main.rs`
**Description:** Moved `permissions: HashSet<Permission>` from `AccountFile` to `CharacterFile`. Rationale: one account can hold an admin character and a regular player character for testing — permissions should be per-character, not per-account. `restore_character()` now loads permissions from char file instead of account file. `#[serde(default = "default_permissions")]` on `CharacterFile.permissions` for backward compat.

### Bootstrap Admin — First Character
**Category:** Code
**Files:** `src/main.rs`
**Description:** First character ever created in the world (detected by `data/characters/` being empty) automatically receives `[player, admin]` permissions. Subsequent characters get `[player]` only. Welcome message notes the admin grant. Eliminates the need to manually edit JSON to get initial Admin access.
